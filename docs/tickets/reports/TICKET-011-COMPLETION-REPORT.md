## 3) 미해결 리스크 / 가정

### 3.1 미해결 리스크

1. 시간 경계(특히 전략1의 15:00 판단과 15:05 체결)에서 데이터 누락 시 손절 체결 불능이 발생할 수 있어, 시뮬레이션 정책(당일 실패/건너뜀)을 구현 단계에서 고정해야 한다.
2. RSI 데이터 정렬 오차 또는 특정 캔들 누락이 잦은 데이터셋에서는 전략3의 매수 기회가 과도하게 줄어들 수 있으므로, 데이터 정합성 경고 로깅 기준을 정의할 필요가 있다.
3. 부동소수 처리 시 경계값(1.0%, 0.2%, 80%, RSI 30) 오차가 발생할 수 있어 Decimal 중심 비교 규칙과 반올림 정책을 테스트로 고정해야 한다.

### 3.2 가정

1. 전략 입력 캔들은 KST 기준 5분 경계로 정렬·정제되어 전달된다.
2. `TradeExecutor`는 `TradeSignal` 스키마를 그대로 수용하며 비용 계산은 실행 계층에서 담당한다.
3. 시뮬레이션 엔진은 전략 예외에 대해 거래일 단위 오류 처리 정책을 보유한다.

## 4) 결론

TICKET-011의 STRATEGY ILD 문서는 LLD/HLD/SRS 기준으로 작성 완료되었으며, 티켓 수용기준 1~6과 품질 요구사항(함수 레벨 인터페이스, 내부 시퀀스/상태전이, 예외 매핑/복구, 구체 수도코드, FR-003~FR-006/NFR-008 추적성)을 모두 충족한다. 남은 리스크는 구현/테스트 단계에서 시간 경계 데이터 누락 정책과 수치 비교 정밀도 기준을 확정하면 관리 가능하다.
