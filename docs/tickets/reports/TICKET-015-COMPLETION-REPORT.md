## 3) 미해결 리스크 / 가정

### 3.1 미해결 리스크

1. SSE 이벤트 발생 빈도가 매우 높은 경우 EventLog 500건 유지 정책에서도 렌더 빈도 증가가 발생할 수 있어, 구현 단계에서 추가 배치 최적화가 필요할 수 있다.
2. 브라우저 네트워크 단절/복구가 반복될 때 서버 버퍼 보존 범위를 초과하면 일부 이벤트 재생 간극이 생길 수 있으므로 `STREAM_REPLAY_GAP` 처리 UX 검증이 필요하다.
3. 결과 보고서 스키마가 1.x에서 필드 확장될 때 프론트엔드 strict parsing 코드가 있으면 호환성 이슈가 발생할 수 있어 계약 테스트가 필요하다.

### 3.2 가정

1. WEBAPI가 `Last-Event-ID` 기반 재전송 규칙을 `ild-webapi-v1.0.0.md`에 맞게 제공한다.
2. REPORT 응답 스키마(`schema_version=1.0`)가 `ild-report-v1.0.0.md` 계약과 일치한다.
3. 프론트엔드 구현 시 Zustand/React Router/EventSource 사용이 가능한 환경을 전제로 한다.

## 4) 결론

TICKET-015의 FRONTEND ILD 문서는 LLD/HLD/SRS 기준으로 작성 완료되었으며, 컴포넌트/스토어/훅 계약, REST/SSE 생명주기·복구 UX, 상태 전이, 성능·사용성 제약, 주니어 구현 수도코드, FR-010~FR-015 및 NFR-003/NFR-004 추적성 요구를 모두 충족한다.
