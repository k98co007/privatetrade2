## 3) 미해결 리스크 / 가정

### 3.1 미해결 리스크

1. `yfinance`는 외부 서비스/비공식 의존성 특성상 응답 스키마 변경 가능성이 있으며, `E-MD-007` 분기 검증을 구현 테스트에서 지속 점검해야 한다.
2. Yahoo 데이터 공백(거래정지/비정상 장 데이터) 시 `NoDataError` 비율이 증가할 수 있어 운영 로그 기반 임계치 모니터링이 필요하다.
3. SQLite 파일 잠금이 빈번한 실행 환경(동시 접근 증가)에서는 `StorageError(E-MD-011)` 대응을 위해 접근 직렬화 정책이 추가로 필요할 수 있다.

### 3.2 가정

1. 입력 심볼은 코스피 포맷(`NNNNNN.KS`)을 따른다.
2. 거래시간 기준은 KST 평일 09:00~15:30이며 5분 경계 데이터만 유효하다.
3. RSI 표준 구현은 Wilder EWM(14기간)을 프로젝트 공통 규격으로 사용한다.

TICKET-010의 MARKETDATA ILD 문서는 LLD/HLD/SRS를 기준으로 작성 완료되었으며, 티켓 수용기준 1~5와 추가 품질 요구사항(SRS FR-001/FR-002 명시 추적, 복구 시나리오, 상태 전이, SQL 구현 상세)을 모두 충족한다. 남은 리스크는 구현/테스트 단계에서 외부 API 변동성 및 저장소 잠금 시나리오를 운영 기준으로 보완하면 해소 가능하다.
