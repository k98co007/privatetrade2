## 3) 미해결 리스크 / 가정

### 3.1 미해결 리스크

1. SSE 재연결 시 서버의 이벤트 재전송 정책(`Last-Event-ID` 지원 범위)이 구현 단계에서 확정되지 않으면 이벤트 중복/누락 가능성이 존재한다.
2. `TradeHistoryTable`의 대량 행 렌더링(장기 확장 시)에서 성능 저하가 발생할 수 있어 가상 스크롤 도입 여부를 ILD 단계에서 검토할 필요가 있다.
3. 브라우저별 미세한 `EventSource` 동작 차이(네트워크 복구 시점)가 있어 E2E 검증 범위를 충분히 확보해야 한다.

### 3.2 가정

1. WEBAPI 모듈은 HLD/LLD에 정의된 REST/SSE 스키마를 안정적으로 제공한다.
2. REPORT 모듈의 응답 스키마(`summary`, `trades`)는 프론트 계약과 호환된다.
3. 메시지 현지화는 한국어 단일 언어로 운영하며 다국어 요구사항은 현재 범위에 포함되지 않는다.

## 4) 결론

TICKET-009의 FRONTEND LLD 문서는 HLD 4.7/5.x/6.x 및 SRS FR-010~FR-015, NFR-003/004 기준으로 작성 완료되었다. 티켓 수용기준과 사용자 작성 요구사항(컴포넌트 인터페이스, store/state machine, REST/SSE 플로우, 상태 전이, NFR 반영, 시퀀스, 수도코드, 추적성, 검증 고려사항, HLD UI 구조 완전 반영)을 모두 충족한다. 남은 리스크는 ILD/구현·테스트 단계에서 SSE 재연결 계약 및 대량 렌더링 전략을 확정하여 해소 가능하다.
